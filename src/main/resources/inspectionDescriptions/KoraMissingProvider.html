Reports parameters in Kora DI injection points (constructors, factory methods) for which no provider is found in the project.

The inspection analyzes parameters of @Component constructors and factory methods in @KoraApp, @Module, and @KoraSubmodule interfaces. For each required dependency type, it searches for a matching provider — a class annotated with @Component or a factory method returning the required type. If no provider is found, the parameter is highlighted with a warning.

Parameters of type All<T> are not checked, since an empty collection is a valid result.

Example:


@Component
public class MyController {
    // Warning: no provider found for 'MyService'
    public MyController(MyService myService) {}
}

This inspection helps catch missing dependency bindings at development time, before they cause errors at application startup.
